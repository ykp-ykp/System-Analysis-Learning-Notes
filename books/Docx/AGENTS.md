# AGENTS.md — books/Docx 目录操作规范

> 本文件适用于 `books/Docx/` 及其所有子目录。
> 当 Agent 在该目录下新增、修改或整理文件时，必须先读取并遵守本文件。

---

## 一、用途说明

`books/Docx/` 仅用于存储 `books/Split Chapters/` 中 PDF 文件对应的小节内容的 Markdown 整理稿。

适用示例：

- `3.1_计算机系统概述`
- `3.2_存储器系统`

---

## 二、文件类型限制

1. `books/Docx/` 及其子目录下只允许新增或维护 `.md` 文件。
2. 新增 Markdown 文件必须与 `books/Split Chapters/` 中的 PDF 章节内容一一对应，不得写入与章节提取无关的内容。
3. 允许按章节建立子目录（如 `books/Docx/3/`、`books/Docx/4/`），但子目录中的文件仍须遵守本规范。

---

## 三、标题层级规范

1. 文件对应的小节标题必须使用二级标题，例如：

```markdown
## 3.1 计算机系统概述
```

2. 若正文包含 `3.1.1` 这类层级，必须使用三级标题，例如：

```markdown
### 3.1.1 计算机系统层次结构
```

3. 若正文包含 `3.1.2.1` 这类层级，必须使用四级标题，例如：

```markdown
#### 3.1.2.1 某子主题
```

4. 不得跳级使用标题；标题层级应与原文编号层级保持一致。

---

## 四、排版规范

1. 避免无意义空格，例如行首多余空格、中文与标点之间异常空格、重复空格。
2. 避免无意义空行；仅在标题、段落、列表等结构之间保留必要换行。
3. 排版应以清晰、紧凑、易读为原则，不得为了“对齐”而插入多余空格或空行。
4. 生成 Markdown 后，必须检查正文是否存在有序编号结构；若原文出现 `1.`、`2.`、`(1)`、`(2)` 等连续编号，Markdown 中也应整理为对应的有序列表，不得保留为普通段落堆叠。
5. 若同一段内容同时存在两级编号，例如外层为 `1.`、内层为 `(1)`，则 Markdown 中应保留其层级关系，分别整理为外层有序列表和内层有序子列表。

---

## 五、来源信息规范

每个 Markdown 文件在正文标题前必须写明来源信息，格式如下：

```markdown
来源：`books/Split Chapters/第3章 计算机系统.pdf`
提取范围：PDF 第 3-13 页。
```

要求：

1. `来源` 必须指向对应的 PDF 文件相对路径。
2. `提取范围` 必须明确 PDF 页码范围。
3. 来源信息应位于正文主标题之前，且表达完整、格式统一。

---

## 六、建议文件结构

建议采用以下结构：

```markdown
<!--
文件名：3.1_计算机系统概述_原文提取.md
创建时间：YYYY-MM-DD HH:mm:ss +08:00
修改时间：YYYY-MM-DD HH:mm:ss +08:00
描述：从对应 PDF 中提取指定小节内容，并整理为 Markdown 文档。
-->

来源：`books/Split Chapters/第3章 计算机系统.pdf`
提取范围：PDF 第 3-13 页。

## 3.1 计算机系统概述

### 3.1.1 ...
1.
2.
```

---

## 七、命名规范

1. 文件名必须采用以下格式：

```text
小节编号_小节标题_原文提取.md
```

2. 推荐按章放入对应子目录，例如：

```text
books/Docx/3/3.1_计算机系统概述_原文提取.md
books/Docx/3/3.2_存储器系统_原文提取.md
```

3. 文件名中不得随意增删章节编号，不得使用“最终版”“整理版”“new”等无语义后缀。

---

## 八、禁止事项

1. 禁止在 `books/Docx/` 下新增非 `.md` 文件。
2. 禁止写入与 `books/Split Chapters/` PDF 小节无关的随笔、总结、脚本或缓存文件。
3. 禁止随意改动来源信息，除非已经核对对应 PDF 并确认原记录有误。
