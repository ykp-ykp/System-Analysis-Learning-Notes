# url-to-md Skill

该 Skill 仅用于根据用户提供的 URL 获取并输出文本信息。它的唯一目的是将网页内容提取为文本，并进行适当的格式美化，直接展示给用户。

## 执行流程

### Step 1：根据用户提供的 URL，抓取网页内容

采用**两级降级策略**抓取网页内容，**不依赖任何第三方 npm 包**。

#### 策略 1（优先）：直接 HTTP 请求

使用 `web_fetch` 工具直接对用户提供的 URL 发起请求，获取并处理页面内容。

- 适用于：绝大多数静态页面、文章、博客。
- 判断成功标准：返回了实质性的文本内容。

#### 策略 2（降级）：使用mcp工具 playwright

当策略 1 失败（如返回空内容、遇到反爬或纯 JS 渲染页面）时：
使用 playwright 相关工具（如 `browser_navigate` 和 `browser_evaluate`）渲染页面并提取正文文本内容。

### Step 2：格式美化与输出

将提取到的原始文本内容进行适当的 Markdown 格式排版（例如整理标题层级、段落、列表、代码块等），使其清晰易读。

最后，直接在对话框中将排版好的完整文本输出给用户。

## 规则约束
- 除了抓取网页并排版输出文本外，**不要**做任何其他事情（如：保存文件、对比现有笔记、生成背诵条目等）。
- 输出的就是网页本身的文本内容（排版后），而不是总结或分析报告。
