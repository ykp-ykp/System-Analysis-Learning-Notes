## 二、目录权限说明

Agent 必须在操作前确认目标目录的读写权限：

| 目录 / 文件 | 权限 | 说明 |
|---|---|---|
| `studyNotes/` | **读写** | 所有笔记输出的唯一合法目录 |
| `vocabulary/` | **读写** | 词汇表，可新增和修改词条 |
| `newTextBook/` | **只读** | 原始教材，仅供参考，禁止任何写操作 |
| `ChangeLog.md` | **追加写**（仅顶部插入） | 变更日志，每次操作后必须更新 |
| `README.md` | **只读** | 项目说明，不得修改 |
| `CLAUDE.md` | **只读** | Claude 专属规范，不得修改 |
| `AGENTS.md` | **只读** | 本文件，不得自行修改 |

## 三、文件输出规范

### 3.1 输出路径（强制）

Agent 生成的所有 Markdown 笔记文件，**必须且只能**保存至：

```
D:\study\System-Analysis-Learning-Notes\studyNotes\
```

任何情况下均不得绕过此规则将文件写入其他路径。

### 3.2 文件命名规范

- 推荐格式：`章节编号_主题名称.md`
  - 示例：`3.1_计算机系统概述_备考笔记.md`
  - 示例：`5.2_数据库范式_考点整理.md`
- 文件名不得包含空格，使用下划线 `_` 分隔
- 文件名不得使用特殊字符：`\ / : * ? " < > |`

### 3.3 文件格式要求

- 编码：UTF-8
- 文件头必须包含元数据注释：

```markdown
<!--
  文件名：xxx.md
  创建时间：YYYY-MM-DD
  最后修改：YYYY-MM-DD
  描述：本文件内容简介
-->
```

## 四、ChangeLog 更新规范（强制）

**任何对项目文件的新增、修改、删除操作完成后，必须立即更新 ChangeLog.md。**

### 4.1 文件路径

```
D:\study\System-Analysis-Learning-Notes\ChangeLog.md
```

### 4.2 写入位置

**必须插入到文件顶部**，确保最新记录排在最上方。禁止追加到文件末尾。

### 4.3 记录格式

```markdown
## [YYYY-MM-DD] 变更类型 — 变更标题

- 变更文件：`路径/文件名.md`（多个文件换行列出）
- 变更内容：本次操作的简要描述（1~3句话，说清楚做了什么、为什么）
- 操作 Agent：[Agent 名称或标识]
```

**示例：**

```markdown
## [2026-05-21] 新增 — 3.1 计算机系统概述备考笔记

- 变更文件：`studyNotes/3.1_计算机系统概述_备考笔记.md`
- 变更内容：新增第3章第1节备考笔记，包含核心概念、高频考点、历年真题及背诵知识点。
- 操作 Agent：Claude
```

### 4.4 变更类型速查

| 类型 | 适用场景 |
|---|---|
| `新增` | 创建了新文件或新增章节/内容 |
| `修改` | 修改了已有文件的内容 |
| `删除` | 删除了文件或内容 |
| `合并` | 将多个文件或内容合并为一个 |
| `重构` | 调整了文件结构或目录组织，内容未变 |
| `修复` | 纠正了错误的知识点、笔误等 |

## 五、Git 提交规范（强制）

**每次变更完成并更新 ChangeLog 后，必须执行以下命令提交：**

```bash
git add .
```


---

## 六、操作前置检查

Agent 在执行任何写操作前，必须先确认：

1. 目标文件是否位于允许写入的目录
2. 是否会影响 `newTextBook/`、`README.md`、`CLAUDE.md`、`AGENTS.md` 等只读文件
3. 当前 git 工作区是否干净（如有未提交变更，需先了解原因）

---

## 七、操作后置检查清单

每次任务结束前，Agent 必须确认以下所有项目均已完成：

- [ ] 所有生成/修改的笔记文件保存在 `studyNotes/` 目录下
- [ ] 文件包含规范的元数据注释头（文件名、创建时间、修改时间、描述）
- [ ] `ChangeLog.md` 已在文件顶部插入本次变更记录
- [ ] 已执行 `git add .`

---

## 八、异常处理原则

遇到以下情况时，Agent 应停止操作并向用户报告，不得自行决策：

- 需要修改 `newTextBook/` 目录内容
- 需要执行 `git push`、`git rebase`、`git reset --hard` 等高风险操作
- 需要删除 `ChangeLog.md` 或 `README.md`
- 发现 git 冲突
- 不确定文件应保存到哪个目录
