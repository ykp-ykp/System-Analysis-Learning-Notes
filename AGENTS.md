# AGENTS.md — AI Agent 通用操作规范

> 本文件适用于所有在 `System-Analysis-Learning-Notes` 项目中运行的 AI Agent。
> **Agent 在执行任何任务前，必须读取并遵守本文件中的全部规范。**
> Claude 专属细则见同目录下的 `CLAUDE.md`。

## 二、目录权限说明

Agent 必须在操作前确认目标目录的读写权限：

| 目录 / 文件               | 权限                           | 说明                                                                                        |
| ------------------------- | ------------------------------ | ------------------------------------------------------------------------------------------- |
| `studyNotes/`           | **读写**                 | 备考笔记、总结类 Markdown 的合法输出目录                                                    |
| `books/Docx/`           | **读写**                 | PDF/章节内容提取类 Markdown 的合法输出目录；操作前必须先读取并遵守 `books/Docx/AGENTS.md` |
| `books/Split Chapters/` | **只读**                 | PDF 来源目录，仅供读取和核对来源信息，禁止写入、删除或改名                                  |
| `vocabulary/`           | **读写**                 | 词汇表，可新增和修改词条                                                                    |
| `ChangeLog.md`          | **追加写**（仅顶部插入） | 变更日志，每次操作后必须更新                                                                |
| `README.md`             | **只读**                 | 项目说明，不得修改                                                                          |
| `CLAUDE.md`             | **只读**                 | Claude 专属规范，不得修改                                                                   |
| `AGENTS.md`             | **默认只读**             | 本文件不得自行修改；仅在项目所有者明确要求时方可调整                                        |
| `.temp/`                | **读写**                 | 用于存放agent在工作时生成的临时文件，用完可清除                                             |

## 三、文件输出规范

### 3.1 输出路径（强制）

Agent 生成或修改的 Markdown 文件，必须根据用途保存到以下合法目录：

| 文件类型                              | 合法目录        |
| ------------------------------------- | --------------- |
| 备考笔记、总结、复盘类 Markdown       | `studyNotes/` |
| PDF/章节原文提取、章节整理类 Markdown | `books/Docx/` |
| 词汇表相关 Markdown                   | `vocabulary/` |

任何情况下均不得将 Markdown 文件写入未授权路径。

### 3.2 子目录规范继承（强制）

当 Agent 操作 `studyNotes/` 及其子目录中的文件时，**必须先读取并遵守**：

```
studyNotes/AGENTS.md
```

当 Agent 操作 `books/Docx/` 及其子目录中的文件时，**必须先读取并遵守**：

```
books/Docx/AGENTS.md
```

当 Agent 操作 `vocabulary/` 及其子目录中的文件时，**必须先读取并遵守**：

```
vocabulary/AGENTS.md
```

若根目录规范与子目录规范同时适用，则应同时满足；如子目录规范更具体，优先执行更具体的约束。

### 3.3 编码规范（强制）

1. 所有新增或修改的 Markdown 规范文件、学习笔记文件、词汇文件默认使用 UTF-8 编码保存。
2. 如需批量处理文件，Agent 必须优先确认不会因编码不一致导致中文乱码。
3. 若终端默认显示异常，Agent 应显式使用 UTF-8 方式读取后再判断文件内容。

## 四、ChangeLog 更新规范（强制）

**任何对项目文件的新增、修改、删除操作完成后，必须立即更新 ChangeLog.md。**

### 4.1 文件路径

```
ChangeLog.md
```

### 4.2 写入位置

**必须插入到文件顶部**，确保最新记录排在最上方。禁止追加到文件末尾。

### 4.3 记录格式

```markdown
## [YYYY-MM-DD] 变更类型 — 变更标题

- 变更文件：`路径/文件名.md`（多个文件换行列出）
- 变更内容：本次操作的简要描述（1~3句话，说清楚做了什么、为什么）
- 操作 Agent：[Agent 名称或标识]
```

**示例：**

```markdown
## [2026-05-21] 新增 — 3.1 计算机系统概述备考笔记

- 变更文件：`studyNotes/3.1_计算机系统概述_备考笔记.md`
- 变更内容：新增第3章第1节备考笔记，包含核心概念、高频考点、历年真题及背诵知识点。
- 操作 Agent：Claude
```

### 4.4 变更类型速查

| 类型     | 适用场景                           |
| -------- | ---------------------------------- |
| `新增` | 创建了新文件或新增章节/内容        |
| `修改` | 修改了已有文件的内容               |
| `删除` | 删除了文件或内容                   |
| `合并` | 将多个文件或内容合并为一个         |
| `重构` | 调整了文件结构或目录组织，内容未变 |
| `修复` | 纠正了错误的知识点、笔误等         |

## 五、操作前置检查

Agent 在执行任何写操作前，必须先确认：

1. 目标文件是否位于允许写入的目录
2. 是否会影响 `README.md`、`CLAUDE.md`、`AGENTS.md` 等只读文件
3. 当前 git 工作区是否干净（如有未提交变更，需先了解原因）

---

## 六、操作后置检查清单

每次任务结束前，Agent 必须确认以下所有项目均已完成：

- [ ] 所有生成/修改的 Markdown 文件保存在授权目录中（如 `studyNotes/`、`books/Docx/`、`vocabulary/`）
- [ ] 若输出为笔记类或原文提取类 Markdown，文件包含规范的元数据注释头（文件名、创建时间、修改时间、描述）
- [ ] `ChangeLog.md` 已在文件顶部插入本次变更记录

---

## 七、异常处理原则

遇到以下情况时，Agent 应停止操作并向用户报告，不得自行决策：

- 需要执行 `git push`、`git rebase`、`git reset --hard` 等高风险操作
- 需要删除 `ChangeLog.md` 或 `README.md`
- 发现 git 冲突
- 不确定文件应保存到哪个目录
