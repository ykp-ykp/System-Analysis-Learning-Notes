# 3.4 指令系统

> 考试定位：综合知识（选择题） | 预计分值：2-3 分 | 出题频率：高

---

## 一、核心概念

### 1.1 指令系统的基本概念

指令是指示计算机执行某些操作的命令，一台计算机的所有指令的集合构成指令系统（指令集）。指令系统是计算机系统中软件与硬件分界面的主要标志。

硬件 vs 软件实现的权衡三因素：速度（硬件快、软件慢）、价格（硬件贵、软件便宜）、灵活性（硬件差、软件好）。

### 1.2 基本指令系统的设计要求

4点要求：完整性（基本指令种类齐全）、规整性（对称性+均匀性）、高效率（使用频率高的指令执行快）、兼容性（向前/向后兼容）。

### 1.3 基本指令的5种类型

数据传送类指令、运算类指令（算术+逻辑+移位）、程序控制类指令（转移+调用/返回+循环控制）、I/O指令（特权指令，管态下运行）、处理机控制和调试指令（管态/用户态）。

### 1.4 复杂指令系统计算机（CISC，Complex Instruction Set Computer）

CISC特点：指令数量多（100～250条）、使用频率悬殊（20%占用80%时间）、多种寻址方式（5～20种）、变长指令、可直接对主存数据操作、微程序控制。

CISC缺陷：80-20规律、VLSI设计负担重、软硬件功能分配问题。

### 1.5 精简指令系统计算机（RISC，Reduced Instruction Set Computer）

RISC通过简化指令使计算机结构更简单合理，减少指令执行周期数。执行时间P=I×CPI×T。RISC的CPI显著低于CISC，速度可提高3～5倍。

RISC特点：指令数量少、寻址方式少（寄存器/立即数/相对）、指令长度固定、硬布线逻辑控制、单周期/流水线、优化的编译器、通用寄存器多（≥32）。

### 1.6 RISC的关键技术

延迟转移技术、指令取消技术、重叠寄存器窗口技术、指令流调整技术、逻辑实现以硬件为主、固件为辅。

## 三、概念对比

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| CISC vs RISC | CISC:指令多、变长、微程序控制、寻址多；RISC:指令少、定长、硬布线控制、寻址少 | C=复杂多杂，R=精简统一 |
| 微程序控制 vs 硬布线控制 | 微程序:灵活但慢（CISC用）；硬布线:快但不灵活（RISC用） | 微程序=软件驱动，硬布线=电路直接控制 |
| 管态 vs 用户态 | 管态可执行特权指令(I/O等)；用户态只能用一般指令 | 管态=管理员模式，用户态=普通用户模式 |
| 直接寻址 vs 间接寻址 | 直接:地址码即操作数地址；间接:地址码指向存储地址的位置 | 直接=直接拿，间接=通过纸条拿 |
| 对称性 vs 均匀性 | 对称:数据存储设备使用和操作码设置对称；均匀:不同数据类型/字长/操作的指令设置同等对待 | 对称=左右一样，均匀=处处一样 |

## 四、历年真题/模拟题

### 模拟题1

**题目：** 以下关于CISC的说法中，不正确的是（ ）。

A. CISC指令数量众多，通常有100～250条
B. CISC以微程序控制为主
C. CISC指令长度固定，译码简单
D. CISC支持多种寻址方式

**答案：** C

**解析：** CISC指令长度是变长的，不像RISC那样固定。变长指令增加了译码复杂性。

---

### 模拟题2

**题目：** 计算机执行一个程序的时间可用P=I×CPI×T表示，RISC速度比CISC快3～5倍的关键在于（ ）。

A. I更小
B. CPI更小
C. T更小
D. I和T都更小

**答案：** B

**解析：** RISC的I长度只比CISC长约20%～40%，T是CISC的约1/3，但CPI（指令平均执行周期数）从4～10降至接近1是关键因素。

---

### 模拟题3

**题目：** 以下不属于RISC特点的是（ ）。

A. 指令数量少
B. 指令长度固定
C. 以微程序控制为主
D. 采用流水线技术

**答案：** C

**解析：** RISC以硬布线逻辑控制为主，CISC才以微程序控制为主。

---

### 模拟题4

**题目：** 设计指令系统时，"各种与指令系统有关的数据存储设备的使用、操作码的设置等都要对称"描述的是（ ）。

A. 完整性
B. 规整性中的对称性
C. 高效率
D. 兼容性

**答案：** B

**解析：** 规整性包括对称性和均匀性，对称性指数据存储设备使用、操作码设置对称（如所有通用寄存器同等对待）。

## 五、背诵知识点

1. 【对比】CISC: 多杂变微程序；RISC: 少简定硬布线(硬布线逻辑控)
2. 【公式】P = I × CPI × T，RISC快3～5倍的关键在CPI大幅降低
3. 【口诀】RISC七大特点：少寻址、固定长、硬布线、单周期流水线、优编译器、多寄存器、LOAD/STORE
4. 【对比】指令设计4要求：完整性（全）、规整性（对称+均匀）、高效率（快）、兼容性（通）
5. 【口诀】5种基本指令：数据传送、运算、程序控制、I/O、处理机控制——"数算程IO机"
6. 【对比】CISC 80-20规律：20%指令占用80%处理器时间，大量复杂指令很少使用
7. 【口诀】RISC关键技术：延迟转移、指令取消、重叠窗口、调整指令流、硬件主+固件辅
