# 3.2 存储器系统 系统分析师备考笔记

> 考试定位：综合知识（上午题） | 预计分值：3~5分 | 出题频率：高

---

## 一、核心概念

存储器系统是计算机体系结构中决定性能的关键子系统。现代计算机采用**层次化存储体系**，自顶向下依次寄存器（Register）、高速缓存（Cache）、主存储器（Main Memory）和辅助存储器（Secondary Storage），形成速度递减、容量递增、单位成本递减的金字塔结构。这一设计的理论基础是**局部性原理**（时间局部性：刚访问的数据短期内可能再次访问；空间局部性：刚访问数据附近的数据可能即将被访问）。系统分析师考试重点考察 Cache 映射与替换算法、RAID 各级别特性、虚拟存储器地址变换和磁盘存取时间计算。

---

## 二、高频考点（最多15条）

1. 存储器层次结构 —— 寄存器→Cache→主存→存的金字塔，局部性原理（时间局部性、空间局部性）【必考】
2. 主存储器分类 —— SRAM（静态随机存取存储器，Static Random Access Memory）与 DRAM（动态随机存取存储器，Dynamic Random Access Memory）对比；ROM（只读存储器，Read-Only Memory）系列：PROM（可编程只读存储器，Programmable Read-Only Memory）/EPROM（可擦除可编程只读存储器，Erasable Programmable Read-Only Memory）/EEPROM（电可擦除可编程只读存储器，Electrically Erasable Programmable Read-Only Memory）/Flash（闪存，Flash Memory）【高频】
3. Cache 地址映射方式 —— 直接映射（Direct Mapping）、全相联映射（Fully Associative Mapping）、组相联映射（Set Associative Mapping）的原理与主存地址划分【必考】
4. Cache 替换算法 —— 随机替换（RAND，Random）、先进先出（FIFO，First In First Out）、最近最少使用（LRU，Least Recently Used）、最不经常使用（LFU，Least Frequently Used）【高频】
5. Cache 写策略 —— 写直达（Write Through，WT）/ 写回（Write Back，WB）、写分配（Write Allocate，WA）/ 非写分配（Not Write Allocate，NWA）【高频】
6. Cache 性能计算 —— 命中率、平均访问时间公式：T_avg = H × T_cache + (1-H) × T_main 【必考】
7. 磁盘存储器性能 —— 存取时间 = 寻道时间（Seek Time）+ 旋转延迟（Rotational Latency）+ 传输时间（Transfer Time）【必考】
8. RAID（独立磁盘冗余阵列，Redundant Array of Independent Disks）—— RAID 0~6 的冗余方式、最少磁盘数、可用容量、容错能力、读写性能对比【必考】
9. 虚拟存储器（Virtual Memory）—— 页式存储管理、段式存储管理、段页式存储管理的原理与地址变换过程【必考】
10. 页式地址变换 —— 页号→页框号的映射，快表（TLB，Translation Lookaside Buffer，转换后备缓冲器）加速地址变换【高频】
11. 页面置换算法 —— 最佳置换（OPT，Optimal）、先进先出（FIFO）、最近最久未使用（LRU）、时钟算法（Clock）【高频】
12. 相联存储器（Associative Memory / CAM，Content Addressable Memory，内容可寻址存储器）—— 按内容而非地址存取，用于 Cache 和 TLB【了解】
13. 存储保护 —— 越界保护（地址界限寄存器）、权限保护（读/写/执行权限位）【了解】
14. 交叉存储器（Interleaved Memory）—— 多体交叉编址，提高并行访问带宽【了解】
15. 磁盘阵列（RAID）容量计算 —— N为磁盘数，C为单盘容量。RAID 0=N×C，RAID 1=N×C/2，RAID 3/5=(N-1)×C，RAID 6=(N-2)×C【必考】

## 三、概念对比

### 3.1 SRAM vs DRAM

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| SRAM（静态随机存取存储器）vs DRAM（动态随机存取存储器） | SRAM 使用双稳态触发器（Flip-Flop）存储，速度快、成本高、容量小、无需刷新，用于 Cache；DRAM 使用电容存储电荷，速度慢、成本低、容量大、需定期刷新，用于主存 | SRAM = "S"uper fast（静态=不用动），DRAM = "D"elay（动态=不断刷新） |

### 3.2 Cache 三种映射方式

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| 直接映射 vs 全相联 vs 组相联 | 直接映射：主存块固定映射到唯一的 Cache 行，冲突高、硬件简单；全相联：主存块可映射到任意 Cache 行，冲突最低、硬件最复杂；组相联：主存块映射到固定组内的任意行，折中方案 | 直接 = "一个萝卜一个坑"；全相联 = "随便坐"；组相联 = "组内自由坐" |

### 3.3 写直达（Write Through）vs 写回（Write Back）

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| 写直达（WT）vs 写回（WB） | WT 写 Cache 时同时更新主存，一致性好但速度慢；WB 仅写 Cache，被替换时才写回主存，速度快但一致性复杂 | WT = "写完就走，主存同步"；WB = "先记账，走时一起算" |

### 3.4 页式存储 vs 段式存储

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| 页式（Paging）vs 段式（Segmentation） | 页式将内存分为固定大小的页框，由硬件自动管理，对用户透明，但会产生内部碎片；段式按逻辑单元（如函数、数据段）分割，大小不固定，方便共享，但产生外部碎片 | 页式 = "切豆腐，大小一致"；段式 = "切菜，按用途分块" |

### 3.5 RAID 各级别对比总表

| RAID级别 | 冗余方式 | 最少磁盘数 | 可用容量（N块盘） | 容错能力 | 读性能 | 写性能 | 典型场景 |
|---|---|---|---|---|---|---|---|
| RAID 0 | 无冗余（条带化，Striping） | 2 | N × C | 无 | 高 | 高 | 视频编辑、临时数据 |
| RAID 1 | 镜像（Mirroring） | 2 | N × C / 2 | 1块 | 较高 | 中等 | 操作系统、日志 |
| RAID 3 | 专用奇偶校验盘 | 3 | (N-1) × C | 1块 | 高 | 较低（校验盘瓶颈） | 大文件流式读写 |
| RAID 5 | 分布式奇偶校验 | 3 | (N-1) × C | 1块 | 高 | 中等 | 文件服务器、数据库 |
| RAID 6 | 双奇偶校验 | 4 | (N-2) × C | 2块 | 高 | 较低 | 金融、关键业务 |
| RAID 10 | 镜像+条带（RAID 1+0） | 4 | N × C / 2 | 每组镜像最多坏1块 | 高 | 中等 | 高性能数据库 |

## 四、历年真题/模拟题

### 模拟题1 —— Cache 命中率与平均访问时间

**题目：** 某计算机 Cache 访问时间为 10ns，主存访问时间为 200ns。某程序执行时 Cache 命中率为 95%，则该存储系统的平均访问时间为（ ）ns。

**答案：** T_avg = 0.95 × 10ns + (1-0.95) × 200ns = 9.5ns + 10ns = **19.5ns**

**解析：** 平均访问时间 = 命中率 × Cache时间 + 失效率 × 主存时间。失效率 = 1 - 命中率。这种计算是上午题中的高频送分题。

---

### 模拟题2 —— Cache 映射方式地址划分

**题目：** 某 Cache 容量为 64KB，采用 4 路组相联映射，块大小为 16B，主存地址为 32 位。则主存地址中，标记（Tag）字段占（ ）位，组索引（Set Index）占（ ）位，块内偏移（Block Offset）占（ ）位。

**答案：**
- 块内偏移 = log₂(16) = **4 位**
- Cache 总行数 = 64KB / 16B = 4096 行
- 组数 = 4096 / 4 = 1024 组（4路组相联）
- 组索引 = log₂(1024) = **10 位**
- 标记（Tag）= 32 - 10 - 4 = **18 位**

**解析：** 主存地址 = Tag + Set Index + Block Offset。组相联映射中，先确认块的偏移量位数（由块大小决定），再计算组数（总行数 ÷ 路数），剩余位即为 Tag。

---

### 模拟题3 —— 磁盘存取时间

**题目：** 某磁盘转速为 7200 RPM（Revolutions Per Minute，转/分钟），平均寻道时间为 5ms，每个磁道有 500 个扇区，每个扇区 512B，则读取一个扇区的平均存取时间约为（ ）ms（忽略传输时间不计时，需考虑平均旋转延迟）。

**答案：**
- 平均旋转延迟 = (1 / (7200/60)) / 2 = (60/7200) / 2 = 8.33ms / 2 ≈ **4.17ms**
- 平均存取时间 = 平均寻道时间 + 平均旋转延迟 = 5ms + 4.17ms = **9.17ms**

**解析：** 旋转延迟是磁盘转半圈的平均时间。7200 RPM → 每秒 120 转 → 每转 8.33ms → 半转约 4.17ms。三个组成：寻道（机械臂移动）+ 旋转延迟（盘片转到目标扇区）+ 传输时间（读数据）。

---

### 模拟题4 —— RAID 容量计算

**题目：** 某存储系统使用 6 块 2TB 硬盘，若配置为 RAID 5，则实际可用存储容量为（ ）TB。若改为 RAID 6，可用容量为（ ）TB。

**答案：**
- RAID 5 可用容量 = (N-1) × C = (6-1) × 2TB = **10TB**
- RAID 6 可用容量 = (N-2) × C = (6-2) × 2TB = **8TB**

**解析：** RAID 5 用 1 块盘的容量存放分布式奇偶校验信息；RAID 6 用 2 块盘的容量存放双奇偶校验。口诀：RAID 5 = N-1，RAID 6 = N-2。

---

### 模拟题5 —— 页面置换 LRU 算法

**题目：** 某进程分配了 3 个页框（Page Frame），页面引用序列为：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5。采用 LRU（最近最久未使用，Least Recently Used）置换算法，共发生（ ）次缺页中断（缺页率 = 缺页次数 / 总访问次数）。

**答案：** 缺页次数 = 次**，缺页率 = 10/12 ≈ 83.3%

**解析（逐步推演）：**
```
访问页：1  2  3  4  1  2  5  1  2  3  4  5
页框1： 1  1  1  4  4  4  5  5  5  3  3  3
页框2： -  2  2  2  1  1  1  1  1  1  4  4
页框3： -  -  3  3  3  2  2  2  2  2  2  5
缺页？ √  √  √  √  √  √  √  ×  ×  √  √  √
```
访问 1、2、3 时页框为空直接缺页；访问 4 时淘汰最久未用的 1；访问 1 时淘汰 2；访问 2 时淘汰 3；访问 5 时淘汰 4；后续 3、4、5 均缺页。共 10 次缺页。

## 五、背诵知识点（不超过15条）

1. 【定义】存储器层次结构自上而下：寄存器→Cache→主存→外存。速度递减、容量递增、成本递减。核心驱动力：局部性原理（时间局部性 + 空间局部性）。
2. 【口诀】SRAM 快贵小（Cache用），DRAM 慢廉大（主存用）。口诀："S=Speed=Cache，D=Delay=Main"
3. 【定义】Cache 直接映射 = 固定座位（块号 mod Cache行数）；全相联 = 随便坐（比较所有 Tag）；组相联 = 组内自由坐（直接映射+全相联折中，现代CPU主流）
4. 【公式】Cache 平均访问时间：T_avg = H × T_cache + (1-H) × T_main。H 为命中率（Hit Rate）。
5. 【公式】Cache 主存地址划分 = Tag（标记字段） + Set Index（组索引/行号） + Block Offset（块内偏移）。
6. 【对比】LRU（Least Recently Used）淘汰最久未使用的 → 符合时间局部性；LFU（Least Frequently Used）淘汰使用次数最少的 → 可能误淘汰即将使用的数据。
7. 【对比】写直达（WT）= 写 Cache 同时写主存（一致性好、速度慢）；写回（WB）= 只写 Cache，替换时写回主存（速度快、需脏位 Dirty Bit）。
8. 【公式】磁盘存取时间 = 寻道时间（机械臂定位）+ 旋转延迟（盘片转半圈的平均时间）+ 传输时间（数据读取时间）。
9. 【口诀】RAID 容量速记：RAID 0 = 全用（N×C），RAID 1 = 一半（N×C/2），RAID 3/5 = 少一块（(N-1)×C），RAID 6 = 少两块（(N-2)×C）。口诀："零全用、一对一、五少一、六少二"
10. 【定义】页式存储：逻辑地址 = 页号P + 页内偏移W。页表存储页号→页框号的映射。引入快表（TLB，Translation Lookaside Buffer）加速地址变换。
11. 【定义】段式存储：按逻辑单元分段（代码段、数据段、堆栈段），每段大小可变。段表存储段基址+段长。便于共享与保护，但会产生外部碎片。
12. 【定义】段页式 = 段式 + 页式：先按逻辑分段，段内再分页。兼具段式的逻辑清晰与页式的内存利用率，但地址变换需两次查表（段表+页表）。
13. 【定义】相联存储器（CAM，Content Addressable Memory）按内容而非地址存取。给定关键字，硬件并行比较所有存储单元，一次性返回匹配项。用于 Cache 的 Tag 比较和 TLB。
14. 【定义】存储保护两种机制：越界保护（地址界限寄存器，检查访问地址是否在合法范围）+ 权限保护（页表/段表项中的读/写/执行权限位）。
15. 【口诀】页面置换算法速记：OPT（最佳置换，Optimal）= 理论最优但无法实现（需要预知未来）；FIFO = 先进先出，简单但可能有 Belady 异常；LRU = 最近最久未用，实际最接近 OPT；Clock = LRU 的硬件近似实现。口诀："OPT 梦、FIFO、LRU 强、Clock 巧"
