# AGENTS.md — studyNotes 目录操作规范

> 本文件适用于 `studyNotes/` 及其所有子目录。
> 当 Agent 在该目录下新增、修改或整理备考笔记、总结、复盘类 Markdown 文件时，必须先读取并遵守本文件。

---

## 一、用途说明

`studyNotes/` 仅用于存储系统分析师备考相关的学习笔记、专题总结、章节复盘等 Markdown 文档。

适用示例：

- `3.1_计算机系统概述_备考笔记.md`
- `数据库事务与并发控制_专题总结.md`

---

## 二、文件类型与命名规范

1. `studyNotes/` 及其子目录下只允许新增或维护 `.md` 文件。
2. 章节型笔记文件名建议采用以下格式：

```text
章节编号_章节标题_备考笔记.md
```

3. 专题型总结文件名建议采用以下格式：

```text
主题名称_专题总结.md
```

4. 文件名中不得使用“最终版”“最新版”“new”“test”等无语义后缀。

---

## 三、元数据注释头（强制）

每个新增的笔记类 Markdown 文件必须在正文前包含统一的元数据注释头，格式如下：

```markdown
<!--
文件名：3.1_计算机系统概述_备考笔记.md
创建时间：YYYY-MM-DD HH:mm:ss +08:00
修改时间：YYYY-MM-DD HH:mm:ss +08:00
描述：系统分析师备考笔记，整理本节核心概念、高频考点、真题提示与背诵要点。
-->
```

要求：

1. `文件名` 必须与实际文件名一致。
2. `创建时间` 与 `修改时间` 必须使用北京时间。
3. `描述` 应简要说明文档用途和内容范围。

---

## 四、正文结构建议

建议采用以下结构组织内容：

```markdown
<!-- 元数据注释头 -->

## 3.1 计算机系统概述

### 核心概念

### 高频考点

### 易错点

### 真题或例题提示

### 背诵要点
```

说明：

1. 若是章节型笔记，主标题应与章节编号保持一致。
2. 若是专题型总结，可将主标题改为专题名称，但正文仍应分区组织。
3. 不得跳级使用标题；标题层级应保持清晰、连续。

---

## 五、排版与内容规范

1. 使用简洁、易读的 Markdown 排版，避免无意义空格和空行。
2. 若内容包含连续编号、口诀、对比项，应优先整理为有序列表或表格。
3. 复盘类内容应尽量区分“知识点总结”“考法提示”“记忆方法”，避免大段堆叠。
4. 除非用户明确要求，否则不得在 `studyNotes/` 中写入与备考无关的草稿、缓存或脚本内容。

---

## 六、编码规范

1. `studyNotes/` 中所有 Markdown 文件默认使用 UTF-8 编码保存。
2. 读取已有笔记时，如终端显示乱码，应先切换为 UTF-8 方式重新读取再判断内容是否异常。
