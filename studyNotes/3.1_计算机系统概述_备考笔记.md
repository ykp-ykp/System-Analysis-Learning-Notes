# 3.1 计算机系统概述 系统分析师备考笔记

> 考试定位：综合知识（上午题） | 预计分值：2~4分 | 出题频率：高

---

## 一、核心概念

计算机系统是由**硬件**和**软件**组成的完整系统。硬件部分包括运算器（ALU，Arithmetic Logic Unit）、控制器（CU，Control Unit）、存储器（Memory）、输入设备和输出设备五大部件，遵循冯·诺依曼体系结构。软件部分包括系统软件和应用软件。系统分析师考试侧重考察计算机系统**体系结构分类**、**指令系统对比**、**存储器层次设计**、**流水线技术**和**系统可靠性**等核心主题，强调原理理解与工程应用的结合。

---

## 二、高频考点（最多15条）

1. 冯·诺依曼结构核心思想——存储程序、五大部件组成、指令与数据统一存放 【必考】
2. Flynn 分类法——按指令流与数据流维度将计算机分为 SISD/SIMD/MISD/MIMD 四类 【必考】
3. CISC 与 RISC 指令系统对比——复杂度、指令长度、寻址方式、编译效率等维度差异 【必考】
4. 流水线（Pipeline，指令流水线）技术——流水线周期、吞吐率、加速比计算 【必考】
5. 存储器层次结构——寄存器、高速缓存（Cache）、主存、外存的速度-容量-成本金字塔 【必考】
6. Cache 地址映射方式——直接映射、全相联映射、组相联映射的优缺点对比 【高频】
7. Cache 替换算法——随机替换（RAND）、先进先出（FIFO，First In First Out）、最近最少使用（LRU，Least Recently Used）、最不经常使用（LFU，Least Frequently Used） 【高频】
8. 磁盘阵列（RAID，Redundant Array of Independent Disks）——RAID 0~6 的特点与适用场景 【高频】
9. 系统可靠性与可用性——平均无故障时间（MTTF，Mean Time To Failure）、平均修复时间（MTTR，Mean Time To Repair）、平均故障间隔时间（MTBF，Mean Time Between Failures） 【高频】
10. 串并联系统可靠性计算——串联系统可靠度相乘、失效率相乘 【必考】
11. 输入输出（I/O，Input/Output）控制方式——程序查询、中断、直接存储器访问（DMA，Direct Memory Access）、通道方式 【高频】
12. 中断（Interrupt）基本概念——中断向量、中断优先级、中断嵌套、中断响应过程 【了解】
13. 总线（Bus）分类——数据总线（DB，Data Bus）、地址总线（AB，Address Bus）、控制总线（CB，Control Bus） 【了解】
14. 多核处理器与并行处理——对称多处理（SMP，Symmetric Multiprocessing）、大规模并行处理（MPP，Massively Parallel Processing） 【了解】
15. 计算机性能指标——主频、每秒百万条指令（MIPS，Million Instructions Per Second）、每秒浮点运算次数（FLOPS，Floating Point Operations Per Second）、基准测试程序 【了解】

## 三、概念对比

| 概念 | 核心区别 | 记忆技巧 |
|---|---|---|
| CISC vs RISC | CISC（复杂指令集计算机，Complex Instruction Set Computer）指令长短不一、寻址方式多、硬件复杂；RISC（精简指令集计算机，Reduced Instruction Set Computer）指令定长、寻址方式少、流水线效率高 | CISC = "大而全"，RISC = "小而精"；C 对 C（Complex → CISC），R 对 R（Reduce → RISC） |
| 冯·诺依曼 vs 哈佛结构 | 冯·诺依曼结构指令和数据共用存储器和总线；哈佛结构指令和数据分开存储、分开总线，并行度高 | 冯·诺依曼 = "一路通行"，哈佛 = "双车道" |
| 直接映射 vs 组相联 |直接映射：主存块→Cache行一一对应，冲突多但硬件简单；组相联：主存块→某组内任意行，折中方案 | 直接 = "固定座位"，组相联 = "组内自由坐" |
| RAID 0 vs RAID 1 | RAID 0 条带化（Striping）无冗余，性能好但安全性低；RAID 1 镜像（Mirroring）全冗余，安全性高但利用率50% | RAID 0 = 速度翻倍风险加倍；RAID 1 = 双份保险 |
| MTTF vs MTBF | MTTF（平均无故障时间）针对不可修复系统；MTBF（平均故障间隔时间）针对可修复系统，MTBF = MTTF + MTTR | MTTF 是"报废前能用多久"，MTBF 是"两次故障间隔多久" |

## 四、历年真题/模拟题

### 真题1（2019年 下半年）

**题目：** 按照 Flynn 分类法，阵列处理器属于（ ）计算机。

**答案：** SIMD（单指令流多数据流，Single Instruction stream Multiple Data stream）

**解析：** 阵列处理器由单一控制单元同时向多个处理单元发出相同指令，处理不同数据，符合 SIMD 特征。

---

### 真题2（2021年 上半年）

**题目：** 某计算机系统由 CPU、存储器、I/O 三部分组成，其可靠性分别为 0.95、0.90、0.85，三者串联，则系统的可靠性为（ ）。

**答案：** 0.95 × 0.90 × 0.85 = 0.72675

**解析：** 串联系统的总可靠度等于各部件可靠度之积。记住"串联可靠度相乘"即可。

---

### 真题3（2020年 下半年）

**题目：** 以下关于 RISC 和 CISC 的说法，正确的是（ ）。

**答案：** RISC 指令长度固定，寻址方式少，有利于流水线执行。

**解析：** RISC 的核心特征就是指令定长、格式规整，使得取指→译码→执行流水线各阶段时长大致相同，流水线效率高。

---

### 模拟题4

**题目：** 某指令流水线分为取指、译码、执行、写回四级，各级耗时分别为 4ns、3ns、5ns、4ns，则该流水线的最大吞吐率约为（ ）。

**答案：** 吞吐率 ≈ 1 / 5ns = 2 × 10^8 条指令/秒

**解析：** 流水线吞吐率取决于耗时最长的一级（瓶颈段），即 5ns，最大吞吐率 = 1 / 瓶颈段耗时。

---

### 模拟题5

**题目：** 两个可靠度均为 0.9 的部件构成并联冗余系统，则该系统的可靠度为（ ）。

**答案：** 1 - (1 - 0.9) × (1 - 0.9) = 1 - 0.01 = 0.99

**解析：** 并联系统的可靠度 = 1 - (各部件失效率之积)。并联意味着"所有部件都失效系统才会失效"。

## 五、背诵知识点（不超过15条）

1. 【口诀】Flynn 分类记忆：SISD=老式单核，SIMD=阵列/GPGPU，MISD=理论存在（脉动阵列），MIMD=多核/集群。口诀："单指令单数据是老爷车，单指令多数据是GPU，多指令多数据是当代标配"
2. 【定义】冯·诺依曼结构的核心：存储程序思想，五大部件（运算器、控制器、存储器、输入设备、输出设备），指令和数据统一存放
3. 【公式】串联系统可靠度：R = R₁ × R₂ × ... × Rₙ
4. 【公式】并联系统可靠度：R = 1 - (1-R₁) × (1-R₂) × ... × (1-Rₙ)
5. 【公式】MTBF（平均故障间隔时间，Mean Time Between Failures） = MTTF（平均无故障时间，Mean Time To Failure） + MTTR（平均修复时间，Mean Time To Repair）
6. 【对比】CISC 指令变长、寻址多、微程序控制；RISC 指令定长、寻址少、硬布线控制、易流水
7. 【公式】流水线最大吞吐率 = 1 / 最长流水段耗时
8. 【公式】流水线加速比 = 顺序执行时间 / 流水线执行时间（理想加速比趋于流水线段数 k）
9. 【定义】Cache 命中率 = Cache 命中次数 / 总访问次数；平均访问时间 = 命中率 × Cache访问时间 + (1-命中率) × 主存访问时间
10. 【对比】直接映射（Direct Mapping）冲突高、硬件简；全相联（Fully Associative）冲突低、硬件复杂；组相联（Set Associative）是折中方案
11. 【口诀】RAID 速记：RAID 0=条带（快但无冗余），RAID 1=镜像（50%利用率），RAID 3/5=奇偶校验（单盘冗余），RAID 6=双盘冗余（最多坏RAID 10=RAID 1+0 组合
12. 【定义】中断向量：存储中断服务程序入口地址的内存单元。中断优先级：多个中断同时请求时 CPU 响应顺序
13. 【对比】程序查询方式 CPU 全程等待；中断方式 CPU 响应请求时切换；DMA 方式（直接存储器访问，Direct Memory Access）由 DMAC 控制总线，CPU 仅初始化
14. 【了解】计算机性能常用单位：MIPS（百万指令每秒，Million Instructions Per Second）= 指令数/(执行时间×10⁶)；MFLOPS（百万浮点每秒，Million Floating Point Operations Per Second）
15. 【口诀】存储器金字塔自上而下：寄存器→Cache→主存→外存，速度递减、容量递增、单位成本递减。口诀："顶快底大，寄存器是皇帝，磁盘是仓库"
